# Реєстр ID та конфігурації інтеграції

Цей файл призначений лише для не секретних ідентифікаторів та технічної
конфігурації.

## Не записувати сюди

Заборонено вставляти:

- Wialon API token;
- поточний `sid`;
- `SUPABASE_SERVICE_ROLE_KEY`;
- `TELEGRAM_BOT_TOKEN`;
- `CRON_SECRET`;
- cookies;
- паролі;
- authorization headers.

Секрети зберігаються лише у Vercel Environment Variables.

### Звіти, які потрібно дослідити зараз

1. `.Отчет по топливу`
   - Template ID: `5`.
   - Resource ID: `2217`.
   - Уже підтверджено основну статистику та таблицю
     `Хронология по топливу`.
   - Додатково знайти тестові дні із заправкою та зливом.

2. `.Поездки`
   - Знайти `reportTemplateId`.
   - Визначити `reportObjectSecId` та тип об'єкта.
   - Перевірити таблиці, колонки та `tableIndex`.
   - Перевірити наявність водія, початкової і кінцевої адреси,
     пробігу, тривалості, швидкості та часу стоянок.

3. `Геозоны`
   - Знайти `reportTemplateId`.
   - Визначити `reportObjectSecId` та тип об'єкта.
   - Перевірити таблиці, колонки та `tableIndex`.
   - Перевірити входи й виходи з геозон, назви геозон, час перебування
     та можливість використання геозон для класифікації маршрутів.

Для кожного звіту зберегти запит `report/exec_report`, результат
`report/apply_report_result` і приклад `report/select_result_rows`.
