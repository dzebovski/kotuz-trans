# Backfill: завантаження даних за багато днів

Інструкція для людини, яка запускає масове завантаження звітів флоту з
Moniterra/Wialon у базу даних.

## 1. Що робить скрипт

Команда `npm run ingest:range` послідовно завантажує дані **за кожен день**
у вказаному діапазоні:

- для **усіх активних машин** (зараз ~24);
- з Moniterra/Wialon у Supabase;
- по одному календарному дню за раз.

Дні, які **вже успішно завантажені**, пропускаються автоматично. Наприклад,
якщо `2026-06-14` і `2026-06-15` уже є в базі, скрипт їх не перезапише
(якщо не додати `--force`).

За замовчуванням скрипт бере **останні 30 днів до вчора** у бізнес-часовому
поясі (`Europe/Kyiv`).

## 2. Перед запуском

1. Переконайтесь, що на комп'ютері встановлено Node.js (версія з
   `package.json` проєкту).
2. У папці проєкту виконайте один раз:

```bash
npm install
```

3. Файл `.env` має бути налаштований (скопіюйте з `.env.example` і заповніть
   значення). **Не надсилайте `.env` іншим людям і не публікуйте в чаті.**

4. Перевірте підключення:

```bash
npm run integrations:test
```

Очікується JSON без помилок, з полями на кшталт `wialonUser` та
`wialonUnitAccess`.

5. **Не запускайте два ingest одночасно** на одному комп'ютері (два термінали
   з `ingest:range` або `ingest:date` паралельно). Це може конфліктувати з
   Wialon і lock у базі.

## 3. Основні команди

### Завантажити останні 30 днів (рекомендовано для першого разу)

```bash
npm run ingest:range
```

### Завантажити свій діапазон дат

```bash
npm run ingest:range -- --from=2026-05-17 --to=2026-06-15
```

Обидві дати включно. Формат: `YYYY-MM-DD`.

### Завантажити іншу кількість днів (замість 30)

```bash
npm run ingest:range -- --days=14
```

Працює лише якщо **не** вказано `--from` і `--to`.

### Перезавантажити один конкретний день

Якщо потрібно оновити лише одну дату:

```bash
npm run ingest:date -- --date=2026-06-13 --force
```

### Надіслати Telegram після кожного дня (зазвичай не потрібно)

```bash
npm run ingest:range -- --send-telegram
```

Для backfill краще **не** вмикати — інакше прийде багато повідомлень.

## 4. Скільки чекати

- Один день (24 машини): приблизно **1 хвилина**.
- 28 днів: приблизно **25–35 хвилин**.

**Не закривайте термінал** до завершення. У процесі з'являються рядки логів
по кожній машині (`vehicle_processed`).

## 5. Як читати результат

У кінці скрипт виводить JSON-підсумок. Головне:

| Поле / статус дня | Значення |
|-------------------|----------|
| `completed` | День завантажено повністю (усі машини) |
| `skipped` | День уже був у базі — пропущено |
| `partial` | Частина машин не завантажилась |
| `failed` | Жодна машина не завантажилась за цей день |

**Все добре**, якщо в підсумку `failed: 0` і немає `partial` (або лише
одиничні `skipped` для уже завантажених днів).

Якщо є `partial` або `failed` — див. розділ «Типові проблеми».

## 6. Типові проблеми

### Помилка мережі або Wialon

Перевірте інтернет і `npm run integrations:test`. Потім перезапустіть
скрипт — уже завантажені дні будуть `skipped`.

### День зі статусом `partial`

Деякі машини не завантажились. Перезавантажте **тільки цей день**:

```bash
npm run ingest:date -- --date=YYYY-MM-DD --force
```

### День зі статусом `failed`

Спробуйте той самий день з `--force`. Якщо повторюється — зверніться до
розробника з датою та текстом помилки з терміналу (без секретів з `.env`).

### Зупинити на першій помилці

```bash
npm run ingest:range -- --stop-on-error
```

Корисно для тестового прогону одного діапазону.

## 7. Перевірка після прогону

1. Запустіть веб-інтерфейс (якщо ще не запущений):

```bash
npm run dev
```

2. Відкрийте в браузері `http://localhost:3000`.
3. Оберіть дату в діапазоні backfill (наприклад `2026-05-20`).
4. Переконайтесь, що є дані по машинах (пробіг, паливо, маршрут).

Альтернатива для розробника: `GET /api/reports/daily?date=2026-05-20`.

## 8. Обмеження

- Дані зберігаються **по календарних днях** (00:00–23:59 за Kyiv). Нічна
  поїздка, що почалась в один день і закінчилась в інший, у звіті дня
  відображається в межах того дня, за який запитувався Wialon — окремого
  «склеювання» між днями цей скрипт не робить.
- Стоянки: у базі є лише **кількість** стоянок за день, без точного часу
  початку/кінця кожної стоянки.
- Telegram за замовчуванням **вимкнений** для batch-завантаження.

## 9. Швидка шпаргалка

```bash
# Перевірка
npm run integrations:test

# Повний місяць (останні 30 днів до вчора)
npm run ingest:range

# Свій діапазон
npm run ingest:range -- --from=2026-05-17 --to=2026-06-15

# Один день повторно
npm run ingest:date -- --date=2026-06-13 --force
```
